## Findings

### P0 — today

**1. Commit and push the restyle.** +989/−414 across 13 files exists only in your working tree. It passes tests, tsc, and my spot-checks (disclaimer intact, no theme bypass, a11y improved). There is no reason for it to stay uncommitted. Suggested split if you want clean history: `theme:` (colors/typography) then `ui:` (screens/components). One commit is also fine — it's one coherent pass.

**2. Fix the stale docs before they misdirect an agent session.** Your own convention is "check what exists before building." The docs now lie:
- `TODOS.md` lists backup/export as deferred → **shipped** (7081f​fab).
- Lists the `getWeeklyVolume` UTC fix as deferred → **shipped** (7884fab).

Any future Claude/Codex session will trust these and either rebuild shipped work or mis-prioritize. 15 minutes, high leverage.

### P1 — next build sessions

**3. Persist the active session (the last data-loss hole).** `useActiveSession` is the only store that isn't persisted. A killed app mid-workout — phone locked 40 minutes, OS reclaims memory, a crash — silently discards every logged set of that session. Mid-workout is precisely when this happens. Fix is cheap and additive: persist `session` + `targets` + `planEmphasis` to MMKV like the other stores; on launch, if a stale active session exists, offer "Resume workout? / Discard". Add a test for rehydrate. This is now the highest-value reliability item in the app.

**4. Add CI.** No `.github/` exists. The 94-test suite only runs when someone remembers. A 20-line GitHub Actions workflow: `npm ci && npx tsc --noEmit && npx vitest run` on push/PR. While you're there: the last commit sets `baseUrl: /Kettlecal` for GitHub Pages, but `dist/` is gitignored and there's no deploy workflow — so deployment is manual or not happening. Add `expo export --platform web` + Pages deploy to the same workflow and the web build becomes self-updating.

**5. Decide the native distribution path.** README promises iOS & Android, but: no `eas.json`, no `ios.bundleIdentifier`, no `android.package`. And `react-native-mmkv` v4 means **Expo Go cannot run this app** — you need a dev build. Decision needed: EAS dev build → personal device install (free, sufficient for a self-tracker) vs TestFlight/internal track (if anyone else will use it). Until this is decided, "iOS & Android" is aspiration, not capability.

### P2 — carried over from the autoplan review, still open (verified in tree)

- **T3** Narrate + undo pain-driven plan changes — only matters if you ever flip `autoAdjust` on; keep deferred until then.
- **T14** Dead code: `RoutineMode` is a single-member union, so the `chooseEmphasisWorkout` branch in `generateWorkout.ts:84` is unreachable. Delete it (and the union), or wire a real KB-primary mode. Dead branches in a 372-line engine file are where bugs hide.
- **T15** High-RPE hold edge: `Math.max(cfg.repMin, best)` can prescribe more reps than the user actually performed when `best < repMin`. Small, but it contradicts the meaning of "hold".
- **T12/T13** Coach-change toast; Progress pain-flags reorder. Polish, fine to leave.

### P2 — new, minor

- **Zero component tests.** Engine coverage is strong; UI has none. For a solo app this is a defensible trade, but `SetLogger` is where training data is born — one interaction test (log a set → store contains it) would protect the core loop through future restyles.
- **`expo-av`** is used only for the EMOM beep and is on a deprecation track (replaced by `expo-audio`). Check the v56 docs (per your own CLAUDE.md) next time you touch the timer; low urgency.
- **`kettlecal.tar.gz`** sits next to the repo as a manual backup. Git + GitHub now supersede it. Delete it or move it out — stale snapshots get restored by accident.
- **CLAUDE.md/AGENTS.md** total 4 lines. Given how much agent-driven work this repo gets, a few more lines (run tests with `npx vitest run`; engine is pure — never add `Date.now`/`Math.random`; all colors via `src/theme`) would raise every future session's floor.
